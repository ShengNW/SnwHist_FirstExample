## 2. 为什么会出现“数值不对”

当前差距不是单点 bug，更像“统计口径 + 训练配方 + 模型来源”三者叠加。

### 2.1 统计口径不完全同构

当前报告主用 `tail10` 训练曲线统计；原文 Table 5/6 是最终评估口径（均值±标准差）。
口径不一致时，数值可比性天然下降。

### 2.2 训练预算与配方偏离论文主设置

当前阶段大量实验使用固定预算快筛（`steps_per_epoch=2400`, `total_steps=144000`）用于定位方向。
这对“趋势筛选”有价值，但对“绝对数值复现（2.7 级 reward）”不够。

### 2.3 TTCT 载入模型与论文原配置未完全锁死

当前使用的 TL 模型路径为项目内 checkpoint。若与论文最终 checkpoint / 数据划分 / 训练轮次不一致，CP 效果会显著漂移。

### 2.4 覆盖面不完整

原文表 5/6 涵盖 `PPO_Lag / CPPO_PID / FOCOPS` 与 `Grid / Goal`。
当前重点投入在 `PPO_Lag + Grid`，尚未形成完整同口径主表。
